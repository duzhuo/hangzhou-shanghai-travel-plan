# 修复导航条选中状态文字颜色问题

## 问题分析
当前导航条选中状态的文字颜色使用的是 `rgba(0, 0, 0, 0.9)`，虽然接近黑色，但从用户提供的截图来看，文字显示为浅灰色，不够清晰。用户希望选中状态的文字保持纯黑色。

## 解决方案
1. **修改选中状态的文字颜色**：
   - 将 `.date-item.selected` 及其子元素的文字颜色明确设置为纯黑色 `#000000`
   - 确保没有其他CSS规则影响文字颜色
   - 保持背景色和文字色的良好对比度

2. **具体修改内容**：
   - 修改 `.date-item.selected` 的文字颜色为 `#000000`
   - 修改子元素 `.date-number`, `.lunar-date`, `.date-location` 的文字颜色为 `#000000`
   - 确保文字颜色不受渐变背景影响
   - 保持良好的视觉效果

3. **修改方案**：
   ```css
   /* 选中日期样式 */
   .date-item.selected {
       background: linear-gradient(135deg, #e0f7fa 0%, #ffffff 100%);
       border: 2px solid var(--td-success-color);
       color: #000000;
       box-shadow: var(--td-shadow-3);
       transform: translateY(-2px) scale(1.02);
   }
   
   /* 确保选中状态下文字颜色为纯黑色 */
   .date-item.selected .date-number,
   .date-item.selected .lunar-date,
   .date-item.selected .date-location {
       color: #000000;
   }
   ```

4. **优化建议**：
   - 调整背景色，使用更浅的颜色，确保文字更清晰
   - 保持边框高亮效果，增强选中状态的视觉识别
   - 确保文字和背景的对比度符合可访问性标准

通过以上修改，导航条选中状态的文字将保持纯黑色，清晰可见，同时保持良好的视觉效果。
